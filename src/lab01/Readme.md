##  Класс Weapon (Оружие)

### Атрибуты (закрытые поля)
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `_name` | str | Название оружия |
| `_weapon_type` | str | Тип (sword, bow, staff, dagger) |
| `_rarity` | str | Редкость (common, rare, epic, legendary) |
| `_level` | int | Уровень (1-10) |
| `_durability` | float | Прочность (0-100) |
| `_damage` | float | Урон (вычисляется автоматически) |

### Атрибуты класса
- `WEAPON_TYPES` - словарь типов оружия
- `RARITY_MULTIPLIERS` - множители урона для разной редкости
- `MAX_LEVEL` / `MIN_LEVEL` - границы уровня
- `MAX_DURABILITY` / `MIN_DURABILITY` - границы прочности

### Свойства (@property)
| Свойство | Геттер | Сеттер | Описание |
|----------|--------|--------|----------|
| `name` | ✅ | ❌ | Только чтение |
| `weapon_type` | ✅ | ❌ | Только чтение |
| `rarity` | ✅ | ❌ | Только чтение |
| `level` | ✅ | ✅ | С проверкой (нельзя понизить) |
| `durability` | ✅ | ✅ | С проверкой (0-100) |
| `damage` | ✅ | ❌ | Вычисляется автоматически |
| `is_broken` | ✅ | ❌ | Логическое состояние |
| `description` | ✅ | ❌ | Красивое описание |

### Магические методы
- `__str__` - красивое описание для пользователя
- `__repr__` - официальное представление для разработчика
- `__eq__` - сравнение объектов

### Бизнес-методы
| Метод | Описание |
|-------|----------|
| `upgrade()` | Повысить уровень на 1 |
| `repair()` | Починить до 100% |
| `attack()` | Атаковать (тратит прочность) |
| `get_status()` | Полный статус оружия |

## Сценарий 1: Нормальная работа с оружием

В первом сценарии демонстрируется создание легендарного меча "Экскалибур" и его использование в игре.

**Создание оружия:** При создании меча автоматически устанавливаются начальные характеристики: уровень 1, урон 30, прочность 100%. Все данные проходят валидацию при создании.

**Улучшение оружия:** Метод `upgrade()` повышает уровень оружия на 1. При повышении уровня автоматически пересчитывается урон. В примере меч был улучшен с 1 до 3 уровня, урон увеличился с 30 до 42.

**Атака оружием:** Метод `attack()` наносит урон врагу и уменьшает прочность оружия. Потеря прочности зависит от редкости: легендарное оружие теряет 5% за атаку. После трех атак прочность снизилась с 100% до 85%.

**Ремонт оружия:** Метод `repair()` полностью восстанавливает прочность оружия до 100%.

## Сценарий 2: Демонстрация валидации (обработка ошибок)

Второй сценарий показывает, как система защищает данные от некорректных значений.

**Ошибки при создании объекта:**
- Попытка создать оружие с пустым именем → ошибка "Имя не может быть пустым"
- Попытка создать оружие с неверным типом ("gun") → ошибка "Неизвестный тип оружия"
- Попытка создать оружие с неверной редкостью ("mythic") → ошибка "Неизвестная редкость"

**Ошибки при изменении уровня:**
- Попытка установить уровень 11 → ошибка "Уровень должен быть от 1 до 10"
- Попытка установить уровень 0 → ошибка "Уровень должен быть от 1 до 10"
- Попытка понизить уровень с 5 до 3 → ошибка "Нельзя понизить уровень оружия"

**Ошибки при изменении прочности:**
- Попытка установить отрицательную прочность → ошибка "Прочность должна быть от 0 до 100"
- Попытка установить прочность выше 100 → ошибка "Прочность должна быть от 0 до 100"

## Сценарий 3: Логические состояния и изменение состояния

Третий сценарий демонстрирует, как объект меняет свое поведение в зависимости от состояния.

**Начальное состояние:** Создается обычный меч с прочностью 100%. Состояние `is_broken = False` (исправно).

**Использование до поломки:** Оружие используется в атаках, прочность постепенно снижается. Обычное оружие теряет 20% прочности за атаку. После 5 атак прочность достигает 0%, оружие переходит в состояние "сломано" (`is_broken = True`).

**Попытка использовать сломанное оружие:** При попытке атаковать или улучшить сломанное оружие выбрасывается ошибка "Невозможно атаковать! Оружие сломано".

**Ремонт оружия:** Метод `repair()` восстанавливает прочность до 100%, оружие возвращается в состояние "исправно" (`is_broken = False`). После ремонта оружием снова можно атаковать.

**Разные состояния объектов:** Демонстрируются три объекта в разных состояниях: исправный легендарный меч, исправный эпический лук и сломанный кинжал.
